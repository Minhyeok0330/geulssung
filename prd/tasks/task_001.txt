# 상세 내용:
PostgreSQL 또는 MySQL을 사용하여 다음과 같은 테이블을 생성합니다:

1. users(사용자):
   - id (기본키, 자동 증가)
   - email (고유값)
   - nickname (고유값)
   - password_hash
   - created_at (생성일시)

2. posts(글):
   - id (기본키, 자동 증가)
   - author_id (users.id 외래키)
   - topic (텍스트)
   - content (텍스트)
   - parent_post_id (posts.id 외래키, null 허용)
   - depth (정수, 0~2: 원글, 반론, 반론에 대한 반론)
   - created_at (생성일시)

3. feedbacks(피드백):
   - post_id (posts.id 외래키)
   - political_score (실수)
   - gender_score (실수)
   - neutrality (실수)
   - feedback_json (AI 피드백 상세 json/텍스트)
   - created_at (생성일시)

4. reactions(반응):
   - id (기본키, 자동 증가)
   - user_id (users.id 외래키)
   - post_id (posts.id 외래키, null 허용)
   - author_id (users.id 외래키, null 허용)
   - type (enum: 'like', 'follow')
   - created_at (생성일시)

5. topics(주제):
   - id (기본키, 자동 증가)
   - keyword (텍스트)
   - description (텍스트)
   - created_at (생성일시)

적절한 인덱스와 제약조건(외래키, 고유값 등)을 구현합니다.

# 테스트 전략:
1. 테이블 생성 스크립트 단위 테스트 작성
2. 샘플 데이터로 외래키 제약조건 테스트
3. 이메일/닉네임 고유값 제약 테스트
4. posts 테이블의 depth(0~2) 제약 테스트
5. 생성일시 필드 자동 생성 확인
6. 샘플 데이터 대량 입력 시 DB 성능 테스트
